## 2. 기획의도
하나의 웹페이지에서 예/적금 상품의 상세정보와 금리비교 등의 기능뿐만 아니라,
환율정보와 주가정보까지 제공하므로써 사용자에게 편리함을 제공한다.
또한, 게시판과 댓글 기능을 이용하여 상품에 대한 사용들의 의견을 나타낼 수 있도록 하고,
지도를 이용하여 은행의 위치들을 검색할 수 있도록 한다.

## 4. 서비스 대표 기능들에 대한 설명 (설계내용)

### 메인페이지
 배너와 환율 계산, 금융 뉴스 그리고 국내증시의 정보를 제공

### 회원 커스터마이징
 회원정보 수정, 닉네임 설정, 생년월일, 성별, 자산, 연봉, 주거래은행의 내용을 담은 회원생성(회원가입) 및 로그인, 로그아웃, 회원정보 수정, 비밀번호 수정, 회원탈퇴 기능 구현

### 커뮤니티
 - 게시글 작성, 수정, 삭제
 - 댓글 작성, 삭제

### 예적금 금리비교
 API를 이용하여 데이터를 받고 상품에 따라서 예적금 이율을 비교 정렬

### 환율 계산기
 API를 이용하여 환율정보를 DB에 저장하고, 유저가 이를 이용할 수 있도록 국가와 금액을 입력받아 계산하여 결과값 도출 및 송출

### 주변 은행 검색
 카카오지도 API를 이용하여 검색이나 클릭시 범위 내에서 위치한 은행 검색 정보제공

### 개인 프로필 페이지
- 작성한 게시글 목록 표시
- 가입한 예/적금 상품 표시
- 예/적금 상품 금리 비교 그래프
- 상세정보 수정


## 5. 알고리즘 설명
 원하는 은행을 선택하면 그 은행에 예적금 이율이 높은 상품을 알려주며, 전체 은행에서 높은 이율 5가지도 알려줘 상품을 비교가능하게 만들어준다.

## 9. 느낌점, 후기

### 김광수
 기능별 역할 분담은 프로젝트 구조의 이해와 효율적인 수정 작업에 크게 기여했습니다. 각 팀원이 자신의 전문 분야에 집중함으로써 프로젝트의 전체적인 진행 속도와 효율성을 높일 수 있었습니다.

### 정범수
 모델링과 기획이 중요하다는 것을 익히 들어 알고 있었지만, 그것을 역할분담 이후에 진행한 것이 이번 프로젝트의 화근이었다고 생각한다. 서로가 생각하는 모델링이 다르고 추구하는 DB구성이 다르기 때문에 미리 약속을 해두지 않는다면 겹치는 부분을 최소화 한다고해도 모델링에서부터 충돌이 날 수 있다는 것을 알게 되었고, 역할분담을 하기 전에 모든 작업을 나누지않고 진행한다고 가정하여 모델링에 대한 협의부터 끝마치고 분담을 해야겠다고 생각했다.
